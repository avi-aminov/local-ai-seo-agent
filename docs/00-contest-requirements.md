# Contest Requirements

Source pages:

- https://dev.to/challenges/google-gemma-2026-05-06
- https://dev.to/page/gemma-4-challenge-2026-05-06-contest-rules
- https://ollama.com/library/gemma4

## Track

Submit this project to:

```txt
Build With Gemma 4
```

The project must build something useful or creative with a Gemma 4 model. Gemma should be central to the product, not only mentioned in the article.

## Dates

- Contest starts: May 6, 2026
- Submissions due: May 24, 2026 at 11:59 PM PDT
- Winners announced: June 4, 2026

## Entry Requirement

The required entry is a published DEV post using the contest submission template.

The post should include:

- project overview
- why Gemma 4 was used
- how the model is used
- screenshots or demo evidence
- GitHub repository link
- clear explanation of what was built

## Judging Criteria

The Build With Gemma 4 prompt is judged on:

- intentional and effective use of the chosen Gemma 4 model
- technical implementation and code quality
- creativity and originality
- usability and user experience

## Model Selection Requirement

The submission should explain why the selected Gemma model fits the project.

For this project, the default target is:

```txt
gemma4:e4b through Ollama
```

Reasoning:

- the product is explicitly about local AI
- SEO analysis benefits from private local inference
- the model can process structured scanner data and return recommendations
- `gemma4:e4b` is stronger than the smallest edge variant while still practical for local development
- local execution supports the contest theme

Document the exact model tag in the README and DEV post.

## Eligibility Notes

- Participants must be 18+.
- Team submissions are allowed for Build With Gemma 4, up to four people.
- One submission can only be eligible for one prompt.
- AI tools are allowed if all rules are followed.
- Open-source code is allowed, but reused code must be credited and the submission must include significant original work.
- Non-English submissions may receive a badge, but prize judging expects English.

## What This Means For Our Project

To be competitive, the project should show:

- a working app, not only documentation
- local Gemma execution
- structured AI output
- a polished and usable interface
- clean Docker-based setup
- clear explanation of engineering decisions

## Winning Angle

The strongest story is:

```txt
A privacy-friendly local SEO audit tool where Gemma turns raw HTML SEO signals into clear, actionable recommendations.
```
